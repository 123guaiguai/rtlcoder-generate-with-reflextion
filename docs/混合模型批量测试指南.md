# AutoChip 混合模型批量测试指南

## 📋 测试配置

本次测试针对**困难组（Prob050-Prob099）**进行优化：

### 1. 迭代次数提升
- **简单组**：3 次迭代（iter0, iter1, iter2）
- **困难组**：6 次迭代（iter0 - iter5）✅ 新增

### 2. 混合模型策略（困难组专用）
- **前 5 次迭代（iter0-iter4）**：使用 **Qwen/Qwen2.5-Coder-32B-Instruct** (Siliconflow API)
  - 成本低，速度快
  - 适合快速尝试和基础修复
  
- **最后 1 次迭代（iter5）**：使用 **GPT-4o-mini** (OpenAI API)
  - 更强的代码生成能力
  - 用于解决疑难问题

---

## 🔑 API Key 设置

由于使用了两个不同的 LLM 服务，需要设置两个 API Key：

### 1. Siliconflow API Key（主要使用）
```bash
export OPENAI_API_KEY="sk-your-api-key-here"
```

### 2. OpenAI API Key（用于最后一次迭代）
```bash
export GPT4O_MINI_API_KEY="your_openai_api_key_here"
```

**重要提示：**
- 如果您没有 OpenAI API Key，可以暂时跳过此步骤
- 脚本会检测 `GPT4O_MINI_API_KEY` 是否存在
- 如果未设置，最后一次迭代将失败，但前 5 次迭代仍可正常运行

## 💰 成本估算

### Siliconflow（前 5 次迭代）
- 定价：$0.002 / 1M tokens
- 假设：Input 200 tokens + Output 50 tokens = 250 tokens/次
- 50 题 × 5 次 × 250 tokens = 62,500 tokens
- 成本：62,500 / 1,000,000 × $0.002 ≈ **$0.000125**

### GPT-4o-mini（最后 1 次迭代）
- 定价：$0.15 / 1M input + $0.60 / 1M output
- 假设只有部分题目需要第 6 次迭代（约 50%）
- 25 题 × 1 次 × (200 input + 50 output) tokens
- Input 成本：25 × 200 / 1,000,000 × $0.15 ≈ $0.00075
- Output 成本：25 × 50 / 1,000,000 × $0.60 ≈ $0.00075
- 总成本：≈ **$0.0015**

### 总成本
- **预计总成本：< $0.002**（约 1.5 美分）
- 相比纯 GPT-4o-mini 方案节省约 **95%** 成本

## 🔍 故障排查

### 问题 1：最后一次迭代失败
**症状**：iter5 显示错误
**原因**：未设置 `GPT4O_MINI_API_KEY` 或 API Key 无效
**解决**：
```bash
export GPT4O_MINI_API_KEY="your_valid_openai_key"
```

### 问题 2：Siliconflow API 调用失败
**症状**：iter0-iter4 全部失败
**原因**：`OPENAI_API_KEY` 未设置或无效
**解决**：
```bash
export OPENAI_API_KEY="sk-your-api-key-here"
```

### 问题 3：编译错误
**症状**：Rank = -1
**原因**：生成的 Verilog 代码有语法错误
**解决**：查看具体题目的日志文件分析错误原因
