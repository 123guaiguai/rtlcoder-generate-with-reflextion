# AutoChip 批量测试使用指南

## 📋 概述

本批量测试系统支持对 VerilogEval 测试集进行自动化、分组、断点续传的批量测试。

### 核心功能

- ✅ **分组测试**：简单组（0-50题）和困难组（50-100题）
- ✅ **断点续传**：已完成的题目自动跳过，支持中断后继续
- ✅ **详细统计**：实时记录通过率、耗时等指标
- ✅ **后台运行**：支持 nohup，防止 SSH 断联
- ✅ **迭代限制**：每道题最多尝试 3 次（iter0, iter1, iter2）

## 🔄 断点续传

### 工作原理

系统会自动检测每个题目的输出目录：
- 如果存在 `log.txt` 且包含 "Iteration rank:"，则认为已完成
- 已完成的题目会自动跳过
- 可以安全中断并重新启动

### 使用场景

1. **SSH 断联**：重新运行脚本即可从断点继续
2. **手动中断**：按 Ctrl+C 停止，稍后重新运行
3. **部分重测**：删除特定题目的输出目录，重新运行只测试这些题目

### 示例

```bash
# 重新运行（自动跳过已完成的题目）
./run_batch_test.sh

# 重测特定题目
rm -rf outputs/batch_tests/easy/Prob001
./run_batch_test.sh  # 只会重新测试 Prob001
```

## 🛠️ 故障排查

### 问题 1：API Key 未设置

**错误信息：**
```
❌ 错误: 未设置 OPENAI_API_KEY 环境变量
```

**解决方案：**
```bash
export OPENAI_API_KEY="sk-your-api-key-here"
```

### 问题 2：Conda 环境未激活

**错误信息：**
```
ModuleNotFoundError: No module named 'openai'
```

**解决方案：**
```bash
source ~/miniconda3/etc/profile.d/conda.sh
conda activate autochip
```

### 问题 3：网络代理问题

**症状：** API 调用超时或连接失败

**解决方案：**
```bash
unset http_proxy https_proxy all_proxy
```

### 问题 4：单道题超时

**症状：** 显示 "⏰ 超时 (>10分钟)"

**原因：** 代码过于复杂或陷入死循环

**解决方案：**
- 检查生成的代码是否有语法错误
- 考虑增加超时时间（修改 `batch_test.py` 中的 `timeout=600`）

### 问题 5：通过率过低

**可能原因：**
1. 模型能力不足
2. 迭代次数太少
3. 题目难度过高

**解决方案：**
- 增加迭代次数：`MAX_ITERATIONS = 3` 或更高
- 更换更强的模型
- 分析失败题目，针对性优化

## 📊 性能估算

### 时间估算

基于之前的测试经验：
- 平均每道题耗时：15-30 秒
- 简单组（49 题）：约 15-25 分钟
- 困难组（50 题）：约 15-25 分钟
- **总计**：约 30-50 分钟

### 成本估算

Siliconflow 定价：$0.002/1M tokens

假设平均每道题：
- Input: 200 tokens
- Output: 50 tokens
- 迭代 3 次

总成本：
```
98 题 × (200 + 50) tokens × 3 次 / 1,000,000 × $0.002 = $0.000147
```

**预计总成本：< $0.001**（几乎免费！）

## 🔧 高级用法

### 自定义分组

修改 `batch_test.py` 中的 `GROUPS` 配置：

```python
GROUPS = {
    "easy": (0, 30),      # 简单组：0-29
    "medium": (30, 70),   # 中等组：30-69
    "hard": (70, 100),    # 困难组：70-99
}
```

### 仅测试特定范围

注释掉不需要的组：

```python
for group_name in ["easy"]:  # 只测试简单组
# for group_name in ["easy", "hard"]:  # 测试全部
```

### 并行测试（实验性）

如需加速，可以使用 GNU Parallel：

```bash
# 安装 parallel
sudo apt-get install parallel

# 并行运行（需谨慎，可能增加 API 费用）
parallel -j 4 python generate_verilog.py -c configs/config_{}.json ::: Prob001 Prob002 ...
```

## 📝 注意事项

1. **API 配额**：确保 Siliconflow API 有足够的配额
2. **磁盘空间**：完整测试需要约 500MB-1GB 存储空间
3. **内存占用**：Python 进程约占 200-500MB 内存
4. **网络稳定性**：建议使用稳定的网络连接
5. **日志管理**：定期清理旧的测试输出，避免占用过多空间
