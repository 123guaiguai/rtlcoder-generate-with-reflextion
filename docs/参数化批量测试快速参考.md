# AutoChip 参数化批量测试 - 快速参考

## 🔑 API密钥配置（硬编码方式）

**重要**: 本项目采用API密钥硬编码方式，无需配置文件或环境变量。

### 已配置的模型信息

#### 1. Siliconflow (硅基流动)
- **Model**: `Qwen/Qwen2.5-Coder-32B-Instruct`
- **Base URL**: `https://api.siliconflow.cn/v1`
- **API Key**: 已硬编码在 `autochip_scripts/languagemodels.py` 中

#### 2. GitHub Models (微软官方)
- **Model**: `gpt-4o-mini`
- **Base URL**: `https://models.inference.ai.azure.com`
- **API Key**: 已硬编码在 `autochip_scripts/languagemodels.py` 中

### 修改API密钥

如需更新API密钥，请编辑以下文件：

```python
# 文件: autochip_scripts/languagemodels.py
# 位置: ChatGPT类的__init__方法中

HARDCODED_API_KEYS = {
    'siliconflow': {
        'api_key': 'sk-your-new-key',  # 修改这里
        'base_url': 'https://api.siliconflow.cn/v1'
    },
    'github': {
        'api_key': 'ghp-your-new-token',  # 修改这里
        'base_url': 'https://models.inference.ai.azure.com'
    }
}
```

**注意**: 
- ✅ 修改后无需任何额外配置，直接运行即可
- ⚠️ 不要将包含真实密钥的代码提交到公共仓库
- 📝 建议定期轮换API密钥以确保安全

## 📋 参数速查

| 参数 | 说明 | 示例 |
|------|------|------|
| `-g` | 组别: easy/hard/all | `-g hard` |
| `-l` | 限制题目数 | `-l 3` |
| `-i` | 迭代次数 | `-i 5` (执行6次) |
| `-k` | 候选数量 | `-k 3` |
| `--mixed-models` | 启用混合模型 | `--mixed-models` |
| `--no-mixed-models` | 禁用混合模型 | `--no-mixed-models` |
| `-n` | 实验名称 | `-n my_exp` |
| `--overwrite` | 覆盖旧数据 | `--overwrite` |
| `--verbose` | 详细输出 | `--verbose` |

**注意**: 不再需要 `-e api_keys.env` 或 `--api-key` 参数

## ⚠️ 常见问题

**Q: 如何更新API密钥？**  
A: 编辑 `autochip_scripts/languagemodels.py` 中的 `HARDCODED_API_KEYS` 字典

**Q: 如何重新开始？**  
A: 使用 `--overwrite` 参数

**Q: 如何停止后继续？**  
A: 直接重新运行相同命令，会自动断点续传

**Q: 输出在哪里？**  
A: `outputs/batch_tests/{实验名}_{时间戳}/`

**Q: 混合模型何时切换？**  
A: hard组默认在iter5切换到GPT-4o-mini（需设置`--iterations 5`或更高）

**Q: 为什么easy组没有混合模型？**  
A: easy组默认`use_mixed_models=False`，如需启用需修改`batch_test.py`中的配置
